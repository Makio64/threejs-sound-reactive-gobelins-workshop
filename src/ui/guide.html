Integration Guide

1. Getting started

The audio engine is dependency-free.
Copy two folders from this repo into your Vite project:

src/sounds/ → your src/sounds/ · the engine public/tracks/ → your public/tracks/ · music (or mic)

On Vite, also copy vite-plugin-vj-tracks.js and register it — it keeps tracks.json in sync, so dropping an .mp3 in public/tracks/ adds it to the playlist:

import vjTracksPlugin from './vite-plugin-vj-tracks.js' export default defineConfig( { plugins: [ vjTracksPlugin() ] } )

Import the analyzer in your code and read the signals each frame:

import Analyzer from '/sounds/Analyzer.js' const audio = new Analyzer() audio.onAudio( a => { a.volume // instant loudness 0..1 a.volumeSmooth // loudness, smoothed 0..1 a.kick // spikes on each kick 0..1 a.kickHard // strong beats only 0..1 a.volumeByFrequency // per-frequency loudness Float32Array(256), 0..1 } )

Run pnpm dev and open your page. It plays the tracks (or mic), so your visuals react on their own.

Press d for a live debug overlay of every signal (spectrum + meters). Drop more .mp3s in public/tracks/, switch with . / ,, and toggle the mic with m.

2. The helper modules

The Analyzer lazy-loads these for you in standalone mode — reach for them only when you want manual control. All three live in src/sounds/.

// AnalyzerDebug- the draggable overlay ('d' toggles it, on by default) import AnalyzerDebug from '/sounds/AnalyzerDebug.js' new AnalyzerDebug( audio ) // .toggle() · .show() · .hide() · .dispose() // SoundPlayer- playlist + audio + keys (m · . · ,), lives at audio.player audio.player?.useTrack( url ) // load a track (also .nextTrack / .prevTrack) audio.player?.useMic() // switch to the microphone // PlayerControl- the "now playing" widget (name = skip · scrub · time) import PlayerControl from '/sounds/PlayerControl.js' new PlayerControl( { getAudioEl, getSource, getTrackName, onSkip, onSeek } )

3. Integrate it here

Add your scene as a local folder and open a pull request — no list to edit. Copy the template, rename it, and edit its index.html:

public/vj-scenes/template/ → public/vj-scenes/your-name/ # edit index.html, save, then open a PR

Save and the dev server auto-detects the folder. Once your PR is merged it joins the loop: the host loads it in an iframe and the same Analyzer auto-switches to receive mode — it reads the show's live audio instead of its own. Nothing in your code changes.
